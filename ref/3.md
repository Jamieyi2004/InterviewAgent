摘   要  I  摘   要  随着物联网与边缘计算技术的快速发展， RISC-V   开源指令集凭借其模块化、可  扩展的设计理念 ， 已成为嵌入式与实时系统领域的重要技术方向 。 然而现有实时操作  系统在移植至   RISC-V   架构时普遍面临内核冗余度高 、 实时响应受限等核心问题 。 本  研究针对上述挑战 ， 介绍了一款专 为   RISC-V6 4   架构设计的轻量级实时操作系统内核 ，  该内核命名为   JaeOS 。  JaeOS   内核通过模块化架构设计，系统化整合了   RISC-V   指令集的可扩展特性与  实时调度需求 ， 在保障核心功能完整性的前提下显著降低了资源开销 ， 开发出高效 的  C   语言实现内核架构。  内存管理子系统实现了结合首次适配分配算法与反向块标记释放策略的分页机  制，优化了物理内存利用率；通过集成   OpenSBI   框架实现硬件抽象，精确控制特权  模式切换与底层设备初始化过程 。 专用   UART0   驱动配备可配置输出缓冲区以支持调  试操作。调度架构采用混合式设计，将时间片轮转与基于优先级的抢占机制相结合 ，  并通过软件触发定时器中断与原子自旋锁实现同步 。 实验验证表明 ， 该内核通过选择  性编译优化显著缩减了体积，同时保持了确定性任务调度能力。  JaeOS   的 模块化架构通过标准化接口解耦硬件依赖关系，确保跨   RISC-V   变体的  平台适应性 。 系统在内存分配效率与实时响应性之间取得平衡 ， 为轻量级内核设计建  立了方法论框架，提供了确定性资源管理的理论洞见及嵌入式系统的实用工程策略 。  该研究证实了在保留操作系统核心功能的前提下实现小 于   130K B   内核的可行性 ，  其可扩展硬件抽象层与中断驱动并发控制机制为基 于   RISC- V   的实时操作系统开发提  供了基础参考框架。  关键词 ： RISC-V ；操作系统内核；实时操作系统；内存管理；任务调度；中断  处理；多任务

Abstract  II  Abstract  Title : Research   and   Implementation   of   an   Operating   System   Kernel   Based   on   R  ISC-V64   Architecture  With   the   rapid   development   of   IoT   and   edge   computing   technologies,   the   RISC-V  open-source   instruction   set   architecture   has   become   a   critical   technological   direction   in  embedded   and   real-time   systems   due   to   its   modular   and   extensible   design   philosophy.  However,   existing   real-time   operating   systems   face   persistent   challenges   when   ported   to  RISC-V   architectures,   including   excessive   kernel   redundancy   and   constrained   real-time  responsiveness. JaeOS   employs   a   modular   architectural   design   that   systematically   integrates   the  extensible   features   of   the   RISC-V   ISA   with   real-time   scheduling   requirements,  significantly   reducing   resource   overhead   while   ensuring   core   functional   integrity.  The   kernel   is   implemented   in   highly   optimized   C   code.   The   memory   management  subsystem   implements   a   paging   mechanism   that   integrates   the   first-fit   allocation   algorithm  with   a   reverse   block-tagging   release   strategy   to   optimize   physical   memory   utilization. The   modular   architecture   of   JaeOS   decouples   hardware   dependencies   via  standardized   interfaces,   ensuring   cross-platform   adaptability   across   RISC-V   variants. 第一章   绪论  1  第一章   绪论  1.1   系统研究背景  RISC-V ，作为一种遵循精简指令集原则的开源指令集架构，其技术原型可追溯  至   2010   年，由加州大学伯克利分校的   David   Patterson   教授团队主导的第五代   RIS C  研究项目 。 相较于   x86 、 ARM   等传统指令集 ， RISC-V   的核心优势在于其开放授权模  式 。 尽管   RISC-V   并非首个开源指令集 ， 但其标准化的指令集分层设计为学术界提供  了模块化的处理器研发范式 。 此外 ， 2019   年 ， RISC-V   基金会联合全球产业界成立 了  RISC-V   国际开源实验室   RIOS   Lab ， 致力于推动该架构从学术探索向工业级应用转型。  现代处理器研发涉及复杂的产业链协同，包括微架构设计、物理实现、 EDA   工  具链以及系统软件生态等多个层级 。 在这些层级中 ， 系统软件层的编译器与操作系统  构成了软硬件协同的关键枢纽 ： 编译器需将高级语言 （ 如   C/Rust ） 转换为符合   RISC-  V   规范的机器码 ， 而操作系统则负责管理硬件资源并为应用程序提供运行时环境 。 由  于   RISC-V   采用模块化指令集设计 ， 开发者可根据应用场景灵活定制   ISA ， 这要求配  套的编译工具链与内核编写过程进行深度适配。目前， GNU-GCC   已实现对   RISC- V  架构的全面支持 ， 同时   RISC-V   官方提供的   ABI   规范与调试接口构建了完整的工具链  支撑体系，显著降低了操作系统内核的开发门槛。  尽管   RISC-V   在学术研究领域显示出显著的价值 ， 但其操作系统内核的研究方法  仍存在局限性：  （ 1 ） 目前 ， 大多数 学术成果依赖于   QEMU   虚拟化平台或   Spike   指令集模拟器进  行验证 ， 这导致了对真实硬件特性的适应性不足 。 例如 ， 在   K210   芯片上 ， 内存时序  的差异可能会在模拟器中引发缺页异常 ， 而现有研究在量化分析这类硬件与软件协同  问题方面尚显不足。  （ 2 ） R ISC-V64   架构的实时操作系统设计目前仍处于初步探索阶段 。 主流移植系  统由于保留了对   x86 、 ARM   架构的历史兼容性，其内核镜像体积和启动时间难以满  足工业物联网对毫秒级响应的需求，且未能充分利用   RISC-V   的定制化优势。

第二章   系统设计  3  第二章   系统设计  2.1   开发语言  RISC-V   架构的内核开发主要采用两种编程语言，即   Rust   和   C 。  以华中科技大学研发的   RustSBI   项目为例 ， 该项目通过   Rust   语言实现了   RISC-V  Supervisor   Binary   Interface ， 为   Rust   内核开发提供了标准化的硬件抽象层 。 该方案借  助   Rust   的内存安全特性 ， 能够有效减少内核中内存泄漏 、 空指针等低级错误的发生 。  JaeOS   选择使用   C   语言进行开发 ， C   语言的指针操作与内存直接映射特性 ， 便于  实现中断上下文保存、页表寄存器配置等硬件相关操作。同时， GCC   等编译器对   C  与汇编的混合编程提供了完善的支持，通过内联汇编与   ABI   兼容性设计，可以无缝  集成   Bootloader 、 中断向量表等汇编模块 。 例如 ， 上下文切换函数   switch_to   需要通过  汇编实现寄存器压栈 / 出栈 ， 而   C   语言可以直接调用此类函数 ， 无需复杂的   FFI   桥接 。  为了避免标准   C   库因动态链接依赖或功能冗余导致内核体积膨胀 ， JaeOS   采取了  极简内核库设计策略：  （ 1 ） 实现必要的底层函数 [6] （如   memcpy/memset ），舍弃文件   I/O   等非核心功  能。  （ 2 ） 针对   RISC-V64   寄存器位宽特性 ， 定义标准化数据类型 （ 如   uint64_t   替代   u  nsigned   long ） 与硬件访问宏 （ 如   read_csr ） ， 确保代码可移植至不同的   RISC-V   平台 。  2.2   开发工具链  JaeOS   的开发工具链见下表   2.1   所示。  表   2 . 1   开发工具链  工具名称   工具版本  Linux   Linux   5.15.167.4-microsoft-standard-WSL2  Qemu   QEMU   emulator   version   9.2.93(v10.0.0-rc3)  Make   GNU   Make   4.3   Built   for   x86_64-pc-linux-gnu  Cmake   cmake   version   3.22.1  Gcc   riscv64-unknown-elf-gcc   version   10.2.0  O bjdump   riscv64-unknown-elf-objdump   version   2.35.1  G db-multiarch   GNU   gdb(Ubuntu   12.1-0ubuntu1~22.04.2)12.1  Readelf   GNU   readelf(GNU   Binutils   for   Ubuntu)2.38
