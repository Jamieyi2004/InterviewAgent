### 一、 系统核心功能规划 (MVP版本)

作为毕设，不要追求大而全，要追求**核心流程的丝滑**。你的系统应该包含以下三个核心阶段：

1.  **准备阶段（配置与简历解析）**
    *   **简历上传与解析：** 用户上传PDF简历，系统自动提取技能栈、项目经验。
    *   **岗位定制：** 用户输入目标岗位（如“Java后端开发”），Agent根据岗位生成针对性的题库。
    *   **面试官人设选择：** 严厉型、温和型、循循善诱型。

2.  **面试阶段（核心交互）**
    *   **实时语音/文字对话：** 模拟真实面试场景。
    *   **多模态交互：** 只有文字太干，最好能有语音合成（TTS）和语音识别（ASR）。
    *   **代码考核（可选高分项）：** 如果是技术面试，提供一个简单的代码编辑器，Agent出题，用户写代码，Agent进行Review。
    *   **追问机制：** Agent根据用户的回答进行深度追问，而不是机械地读题。

3.  **复盘阶段（评估报告）**
    *   **多维度打分：** 技术深度、沟通能力、逻辑思维。
    *   **详细点评：** 哪道题答得不好，参考答案是什么，建议如何改进。

### 二、 系统架构设计

我们将系统分为三层：**前端交互层**、**后端业务层**、**AI算法服务层**。

#### 1. 架构图解 (概念)

```mermaid
[用户浏览器] <--> [前端 (Next.js)] <--> [后端 (FastAPI/Python)] <--> [LLM 服务 (OpenAI/Kimi/本地模型)]
                                      |
                                   [数据库 (PostgreSQL/SQLite)]
                                      |
                                   [向量库 (ChromaDB/Faiss) - 可选，用于RAG]
```

#### 2. 技术栈选型 (追求“迅捷”与“展示效果”)

这是我为你推荐的**2026年主流且高效**的技术栈组合：

**前端 (Frontend): Next.js + Tailwind CSS + ShadcnUI**
*   **理由：**
    *   **Next.js (React):** 毕设演示神器，SSR渲染速度快，Vercel一键部署。
    *   **Tailwind CSS:** 写样式极快，不用切图。
    *   **ShadcnUI:** 目前最火的组件库，复制粘贴代码即可，界面极其精美，非常适合做Chat界面。
    *   **状态管理:** Zustand (比Redux简单一万倍)。

**后端 (Backend): Python (FastAPI)**
*   **理由：**
    *   **Python:** 做AI系统的不二之选，方便调用LangChain等库。
    *   **FastAPI:** 性能极高，自带Swagger文档（写论文截图神器），代码量少，支持异步（WebSocket做实时对话很关键）。

**AI/算法核心 (The "Agent" Brain): LangChain + LLM API**
*   **模型选择：** 建议直接调用API（如 **DeepSeek**, **Kimi**, **OpenAI**），不要自己训练模型（耗时且显卡要求高）。如果是“算法部分”需要体现工作量，可以侧重于 **Prompt Engineering（提示工程）** 和 **RAG（检索增强生成）**。
*   **框架:** **LangChain** 或 **LlamaIndex**。这是构建Agent的标准框架，用来管理对话历史（Memory）和工具调用（Tools）。

**数据库 (Database):**
*   **主库:** SQLite (开发阶段) -> PostgreSQL (部署阶段)。用 **Prisma** 或 **SQLAlchemy** 做ORM。
*   **向量库 (Vector DB):** ChromaDB (轻量级，本地文件存储，无需启动Docker容器)。

### 三、 核心算法部分实现思路 (毕设亮点)

作为“算法部分”的负责人，你不能只调API，你需要设计Agent的**思维链 (Chain of Thought)**。

#### 1. 简历解析与画像生成 (RAG预处理)
*   **技术：** 使用 `PyPDF2` 解析简历文本 -> 使用 LLM 进行结构化提取（输出JSON格式） -> 存入数据库。
*   **算法点：** 设计一个Prompt，让LLM根据简历内容提取关键词，并根据这些关键词去向量库里检索相关的面试高频题（RAG）。

#### 2. 面试官 Agent 的状态机设计
不要让Agent只是简单的 `Input -> Output`。设计一个简单的有限状态机 (FSM)：
*   **阶段0 (开场):** 寒暄，自我介绍。
*   **阶段1 (八股文):** 基于简历关键词提问基础知识。
*   **阶段2 (项目深挖):** 针对简历里的项目经历，使用STAR法则进行追问（"你在这个项目中遇到的最大难点是什么？"）。
*   **阶段3 (代码/算法):** 发送一道LeetCode题目（可选）。
*   **阶段4 (结束):** 总结陈词。

*你需要编写代码逻辑来控制这个流程的跳转。*

#### 3. 实时语音交互 (WebSocket)
*   **前端：** 使用浏览器原生 `Web Speech API` (免费) 做语音转文字。
*   **通信：** 使用 WebSocket 保持长连接，实现打字机效果（流式输出）。
*   **后端：** 接收文字 -> LLM生成回复 -> 调用 TTS API (如Edge-TTS 免费且效果好) -> 返回音频流给前端播放。

### 四、 开发路线图 (Timeline)

**第一周：环境搭建与Hello World**
1.  初始化 Next.js 项目，搭建聊天界面（左边对话框，右边简历/代码区）。
2.  初始化 FastAPI，跑通一个简单的 `/chat` 接口，调用 LLM API 返回数据。

**第二周：核心业务逻辑**
1.  实现简历上传接口，解析PDF内容。
2.  设计 Prompt Template（提示词模板），例如：`你是一个资深Java面试官，现在你要根据这份简历内容：{resume_content}，对候选人进行面试。`

**第三周：Agent 逻辑增强**
1.  引入 LangChain 的 Memory 模块，让AI记住之前的对话。
2.  实现“追问”逻辑：判断用户的回答是否完整，不完整则继续追问。

**第四周：完善与多模态**
1.  接入 TTS (文字转语音) 和 ASR (语音转文字)。
2.  生成面试评估报告。

**第五周：论文与优化**
1.  美化UI。
2.  截取Swagger文档、架构图、对话截图用于论文。

### 五、 毕设加分项 (如果你学有余力)

1.  **跨会话记忆（用户画像追踪）：** 系统能跨越多次面试记住用户的表现，实现个性化出题与成长追踪。
    *   **核心思路：** 每次面试结束后，从评估报告中提取用户的薄弱知识点、擅长领域、平均得分等信息，存入 `user_profiles` 用户画像表。
    *   **数据模型（预留）：**
        ```sql
        CREATE TABLE user_profiles (
            id               INTEGER PRIMARY KEY AUTOINCREMENT,
            user_id          TEXT NOT NULL,
            weak_topics      TEXT,    -- JSON: 薄弱知识点列表，如 ["并发","数据库索引"]
            strong_topics    TEXT,    -- JSON: 擅长领域列表，如 ["Spring","设计模式"]
            interview_count  INTEGER DEFAULT 0,
            avg_score        REAL,
            last_report_summary TEXT, -- 上次面试的LLM摘要
            updated_at       DATETIME DEFAULT CURRENT_TIMESTAMP
        );
        ```
    *   **应用场景：**
        *   "上次面试你的并发知识较薄弱，这次我们重点考考这块"
        *   生成"成长曲线"报告，展示多次面试的进步轨迹
        *   根据历史薄弱点自动调整出题权重
    *   **实现策略：** 面试结束 → LLM总结本次表现 → 与历史画像合并 → 下次面试时注入Prompt上下文
    *   **优先级：** 中期答辩暂不实现逻辑，仅预留数据模型；终期答辩作为亮点功能完整实现。

2.  **情绪分析：** 在后端分析用户的语音语调或文字情感，如果用户表现紧张，Agent可以说："别紧张，我们换个轻松的话题。"
3.  **实时代码运行：** 接入一个可以在线跑代码的沙箱（如 `Judge0` API），用户写完代码，Agent不仅能看代码，还能真的运行测试用例。
