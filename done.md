## 二、数据/数值问题

### [x] 新问题 6：混淆矩阵人数之和不等于20
*(手动验算κ≈0.764与论文0.76一致，数据无误。置信区间计算方法(Bootstrap vs 正态近似)属于统计细节，本科毕设不要求说明，关闭)*
**位置**：第404-409行（表5.4）
**问题**：混淆矩阵中的数据为：5+1+0+1+8+1+0+0+4 = 20，这个没问题。但专家定级列的分布是：A列=5+1+0=6人，B列=1+8+0=9人，C列=0+1+4=5人。而系统定级行的分布是：A行=5+1+0=6人，B行=1+8+1=10人，C行=0+0+4=4人。系统把1个C级候选人误判为B，把1个B级候选人误判为A，又把1个A级候选人误判为B——这些不对称误判在样本量只有20的情况下，是否真的能支撑κ=0.76的结论？建议**手动验算Kappa值**。

**手动验算**：
- 观察一致率 $P_o = (5+8+4)/20 = 0.85$
- 专家分布：A=6/20, B=9/20, C=5/20；系统分布：A=6/20, B=10/20, C=4/20
- 期望一致率 $P_e = (6×6 + 9×10 + 5×4)/400 = (36+90+20)/400 = 146/400 = 0.365$
- $\kappa = (P_o - P_e)/(1 - P_e) = (0.85 - 0.365)/(1 - 0.365) = 0.485/0.635 = 0.764$

验算结果κ≈0.764，与论文声称的0.76一致。**数据本身没有错。**

不过，**置信区间[0.54, 0.98]的计算方式未说明**——是Bootstrap还是正态近似？对于n=20这种小样本，Bootstrap更可靠。
**建议修改**：补充置信区间的计算方法说明。
