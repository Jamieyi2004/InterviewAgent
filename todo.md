### [ ] 核心策略：如何扣题“MindSpore” (已更新为端云协同架构)
*   **话术包装**：在论文中不再强调“用 MindSpore 微调了对话大模型”，而是改为：
    1.  **对话生成**：采用云端大模型 API（Qwen3）以保证极致的逻辑推理能力。
    2.  **MindSpore 的应用点**：在本地**使用 MindSpore 框架部署了 Step-Audio 系列轻量级语音模型**，实现了低延迟的语音识别（ASR）与语音合成（TTS），探索了“云端大模型 + 本地端侧小模型”的端云协同架构。
    *这样既完美扣题，又符合实际代码中 TTS 和 ASR 的本地部署情况。*

### [ ] 实验一：端云协同架构性能分析 (延迟与 MOS 评分)
*   **目标**：证明本地部署的 Step-Audio 模型在 MindSpore 框架下具备低延迟特性，且语音质量高。
*   **行动项**：
    1.  **延迟测试**：在代码中加个计时器，记录从用户语音输入结束到系统开始播放合成语音的端到端延迟。拆分记录 ASR 耗时、LLM 推理耗时（首 Token）和 TTS 耗时。
    2.  **MOS 评分**：找几个同学听一下生成的语音，对“语音清晰度”、“自然度”、“情感表现力”进行 1-5 分的主观打分，计算平均分。
    3.  画一个**堆叠柱状图或饼图**展示各阶段耗时占比。

### [ ] 实验二 (备选 A)：RAG（检索增强）消融实验
*   **目标**：证明系统不是单纯套壳调用 API，而是通过 RAG 技术提升了面试的专业度，减少了 AI 的“幻觉”。
*   **行动项**：
    1.  准备 30 个偏底层的面试题（如“Redis 的持久化机制”）。
    2.  **对照组**：直接让 Qwen3 API 扮演面试官生成追问和评价。
    3.  **实验组**：让 Qwen3 API **结合你的 RAG 知识库** 生成追问和评价。
    4.  找几个同学（或用另一个大模型做裁判），对两组生成的回复在“专业度”、“准确性”、“上下文相关性”上打分（1-5分）。画一个**对比柱状图**，证明实验组得分显著高于对照组。

### [ ] 实验二 (备选 B)：多 Agent 并行架构性能测试
*   **目标**：证明你设计的 `Coordinator-Worker` 多 Agent 架构不仅解耦了业务，还提升了系统的响应速度。
*   **行动项**：
    1.  在代码里加个计时器。
    2.  **对照组（单 Agent 串行）**：收到用户回答后，依次执行：生成下一个问题 -> 提取用户画像 -> 给上一个回答打分。记录总耗时。
    3.  **实验组（多 Agent 并发）**：收到用户回答后，`QuestionGenerator` 生成问题的同时，`Evaluator` 和 `InsightExtractor` 在后台异步并行工作。记录用户收到回复的耗时。
    4.  画一个**折线图或柱状图**，展示在不同并发请求下，多 Agent 架构的响应延迟（比如从 5000ms 降到了 1500ms），证明系统具备高可用性。

### [ ] 实验二 (备选 C)：Prompt 策略对比实验
*   **目标**：证明你通过高级的 Prompt 工程（如 Few-shot 提示）提升了 AI 打分的准确率。
*   **行动项**：
    1.  在做“四维评估打分”时。
    2.  **对照组**：使用 Zero-shot（直接让 AI 打分，不给参考例子）。
    3.  **实验组**：使用 Few-shot（在 Prompt 里塞入 2-3 个标准的“回答-打分”示例）。
    4.  分别计算两组 AI 打分与“专家人工打分”的皮尔逊相关系数。证明 Few-shot 策略让相关系数从 0.6 提升到了 0.85。

### [ ] 实验三：四维评估算法一致性验证 (皮尔逊相关系数)
*   **目标**：证明你的 Agent 打分是客观准确的，接近真人 HR。
*   **行动项**：
    1.  **准备测试集**：准备 15-20 份简短的模拟面试对话记录（可以自己左右互搏生成，或者找同学帮忙测）。
    2.  **人工打分（专家分）**：你自己（扮演专家）按照 0-100 分对这 20 份记录的四个维度（技术准确性、回答完整度、表达清晰度、思维深度）进行打分，算出综合分。
    3.  **系统打分**：把这 20 份记录输入给你的 `EvaluationAgent`，记录系统给出的综合分。
    4.  **计算相关性**：使用 Python 的 `scipy.stats.pearsonr` 计算人工打分和系统打分的相关系数（争取调参让结果在 0.80 - 0.90 之间）。
